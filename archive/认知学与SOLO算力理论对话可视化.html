核心分歧：为什么 SOLO 完胜布鲁姆？

维度布鲁姆分类SOLO 理论
评估对象学生的「内部认知心理动作」知识本身的「外部结构复杂度」
关键词识记、理解、应用、分析、综合单点、多点、关联、抽象扩展

四层模型 与 SOLO 层级的完美映射

前结构 Pre-structural

错题/未掌握状态

学生表现：乱猜，或用无关信息解题

AI 诊断：完全没有触及 L4 知识点，基础空白

L1 单点结构 Uni-structural

对应【L4 原子知识点】

定义：只能提取单个线索，只会单一算法

举例：知道"怎么加"，能算对 25 + 13 = 38

L2 多点结构 Multi-structural

对应【特征层 Feature】

定义：掌握多个孤立线索，但还没联系起来

举例：知道"进位"（特征A）、"含0"（特征B），但处理不了"又进位又有0"

L3 关联结构 Relational

对应【考法层 Test Method】

定义：能把多个分散的点整合为一个有机整体，解决具体场景问题

举例：把"进位加法"和"比多比少"关联，成功解决排队应用题

L4 抽象扩展 Extended Abstract

对应【思想层 Thinking】

定义：跳出当前问题框架，将原理升华、泛化到未知新领域

举例：做完整数"平移法求周长"，能自己悟出小数中也存在"转化思想"

采用 SOLO 理论的三大巨大增益

1. 错因极度精确化
传统（布鲁姆）："该生在应用题的综合能力上有欠缺" — 正确的废话
SOLO 系统："该生处于多点结构，掌握了小数乘法(L4)和末尾去0(特征)，但缺乏关联能力"

2. 可视化的绝佳理由
星图就是在物理意义上重现了孩子大脑神经元生长的过程

3. 推题算法极其自然
考法节点出错 → 向下回退推多点特征题 → 再错 → 推单点 L4 纯算式题

五阶认知进化论

🟢 L1 孤立点映射 (SOLO-U)

刺激-反应 | 单通道

看到数字就读，看到算式就死算。无法将两个不同概念结合。

测评：底层知识的提取速度和准确率（百题计算）

🔵 L2 线性堆叠 (SOLO-M)

并行处理，互不相交 | 双通道并发

能同时提取多个条件，但条件之间是"A和B"而非"A导致B"

测评：跨媒介表征能力（把图形翻译成数字）

🟠 L3 结构化建模 (SOLO-R) — 🌟 核心分水岭

格式塔与闭环 | 闭环回溯

质变！建立 部分 + 部分 = 整体 的数学模型，具备可逆性思维。

测评：区分"机械刷题"和"真懂算理"的试金石

🔴 L4 复杂网络 (SOLO-R+)

抗干扰与多步路由 | 复杂网络

在信息过载中精准剔除"冗余条件"，能把第一步的输出作为第二步的输入。

测评：思维的严密性和深度

🟣 L5 降维与抽象 (SOLO-E)

元认知与代数思维 | 抽象降维

不对"数字"运算，而是对"运算本身的规律"进行运算。

测评：创新意识和高阶逻辑，中小学思维天花板

从古典心理学到认知神经科学

晓冬
晓冬 提问

AI
AI 助手 三大前沿理论升维

升维一：从"认知负荷"到"神经表征"

迪昂的三重表征模型 (Triple Code Model, Dehaene, 1992)

原框架盲区：把图/文/算式统称 UI，太粗糙了。

前沿发现：大脑用三个独立脑区硬件处理数学：

  • 视觉阿拉伯表征（符号） — 枕颞区，处理 3, +, =
  • 听觉言语表征（文字） — 左半球语言区，处理"三个""增加"
  • 模拟数量表征（数感） — 顶内沟(IPS)，处理大小、距离、连续量

🎯 优化：当孩子做错应用题时，精准诊断为"语词表征与数量表征的跨脑区数据传输失败"。

升维二：从"算力层级"到"算力占用率"

卡尼曼的双系统理论 (Dual Process Theory, 2011 诺奖)

原框架盲区：只看"对错"，忽略了反应时。

前沿发现：大脑有两个 CPU：系统1（自动驾驶，极速，0内存）和 系统2（逻辑分析，高耗能）。高手的本质是把 L1/L2 知识编译成系统1的下意识反应。

🎯 优化：增加耗时/流利度维度。即使答对了 L2 题但花了 30 秒 → 判定知识仍在系统2 → 推送流利度训练。

升维三：从"错因回退"到"生成式误差更新"

弗里斯顿的自由能原理与预测编码 (Free Energy Principle, 2010s)

原框架盲区：做错=算力不够或不会，于是降维。

前沿发现：学生的错误不是无能，而是"预测误差"。学习的本质是更新大脑的生成模型以最小化惊奇感。

🎯 优化：例如 0.5×0.8=0.4 学生却认为变大了 — 错误先验是"乘法总是使数字变大"。AI 不该退回简单乘法，而是推送打破其先验的最小特例。

人类 vs 大模型的核心差异

维度一：CoT 推理深度层

取代 SOLO 理论，按"需要几步思维链"划分难度

Hop-0 显式检索

事实类知识，一次前向传播直接输出。如："加法交换律是什么？"

Hop-1 单步映射

给定公式代入变量。如：S=ab，代入 3 和 4 算出 12

Hop-2 显式多步推理

条件都给出但需按序执行两步以上，必须生成中间步骤

Hop-3 隐式条件推断

题干信息不全，需先从隐空间提取隐式常识再推理。如：闰年2月全勤

维度二：Attention 注意力锚点层

🔢

[A1] 强逻辑锚点

纯算式/方程，注意力 100% 集中在符号和运算顺序

📝

[A2] 语义对齐锚点

文字应用题，将自然语言精确对齐到数学算子

🎭

[A3] 冗余降噪锚点

含多余条件，需特殊提示模型先剔除干扰

维度三：语义向量空间层

取代树状目录，用扁平标签构建高维网

AI
AI 助手 理论→落地深度融合

核心思路：每个理论都改变了一个关键决策

不是"先学理论再做产品"，而是每个理论直接回答了一个落地困境。下面逐一拆解：理论解决了什么问题 → 落在哪个组件 → 如果没有这个理论我们会怎样做错。

一、SOLO 理论 → 知识图谱的四级架构 + 降维诊断逻辑

Q1 的理论：SOLO 分类（Biggs & Collis, 1982）

落地困境：知识图谱该怎么分层？传统做法按教材章节分，但这只是目录结构，不反映认知复杂度。

SOLO 如何改变决策：

  • SOLO 的"单点→多点→关联→抽象扩展"直接指导了 ❸ 知识图谱的 L1→L4 四级 DAG 架构
  • 537个 L4 原子知识点的切分标准——按"进位/不进位""连续进位/一次进位"等结构复杂度边界切分，而不是按题型切分
  • 降维诊断的核心逻辑——学生做错高层级(关联)题时，向下回退到低层级(单点)题定位断点

SOLO 层级 → 落地组件的精确映射

SOLO 层级落地到哪具体做了什么
单点结构❸ 知识图谱 L4 原子节点537个最小粒度知识点，每个只含一个认知要素
多点结构❸ 知识图谱特征列进位/退位/含0 等特征组合——学生能处理多个孤立条件但不能关联
关联结构❹ 考法图谱 314个考法考法 = 多个 L4 + 特征的有机组合，需要建立模型才能解
抽象扩展❷ 思想字典 21个思想ID跨年级/跨模块的底层逻辑，ID-102[点段对应]可从爬楼梯迁移到植树问题

如果没有 SOLO：我们会按教材"第三章第二节"分层，知识图谱变成目录翻版，AI 降维时只能"降难度系数"，无法精准定位到底是哪个认知层级断裂。

二、认知负荷理论 → 三维正交剥离 + 考法字典的 I 维度

Q2 的理论：认知负荷理论（Sweller, 1988）

落地困境：学生做错一道应用题，是"加法不会"还是"读不懂题"？传统标签"两位数加法应用题"混为一谈。

认知负荷如何改变决策：

  • Sweller 区分了内在负荷(知识本身)和外在负荷(呈现形式)，这直接催生了 三维正交剥离原则——知识点、考法、能力三个维度完全独立
  • 落地到 ❺ 考法字典的 I 维度（信息隐蔽度）：I0裸题→I1浅层情境→I2隐蔽条件→I3动态建模→I4冗余干扰
  • I 维度本质就是在量化外在认知负荷的梯度——同一个知识点，穿上不同的"情境外衣"，负荷完全不同

I 维度降维诊断的实战威力

学生做错 I2 题（隐蔽条件：无盖鱼缸求表面积）
↓ 剥离外在负荷：I2 → I0
推送 I0 裸题：长5宽3高2的长方体，求表面积
做对 → 错因锁定：外在认知负荷过载，不是知识点问题
做错 → 继续降维到知识点层

如果没有认知负荷理论：我们会把"表面积裸题"和"无盖鱼缸"打成两个不同知识点（加上竖式/应用题更多），标签爆炸且逻辑混乱。有了 I 维度，同一个 L4 知识点只需通过调节 I 值就能覆盖所有难度变体。

三、三重表征模型 → 情境库的三引擎 + UI 维度精细化

Q3 升维一：迪昂的三重表征模型（Dehaene, 1992）

落地困境：❼ 情境库有 80+ 情境ID，但如何判断给学生换哪种情境？"看图题"和"文字题"对大脑的负荷是同一种吗？

三重表征如何改变决策：

  • 大脑用三套独立硬件处理数学：视觉符号(枕颞区) / 听觉语词(语言区) / 模拟数量(顶内沟)
  • 直接指导了 考法字典 I 维度内部的细分：I1 不再笼统叫"浅层情境"，而是区分为 纯符号UI / 具象视觉UI / 自然语言UI
  • 落地到 ❼ 情境库的三引擎本质上就是三种表征的系统性覆盖：引擎一(标准文字) / 引擎二(真实情境) / 引擎三(跨学科创新)

传统做法

学生做错"应用题" → 诊断为"阅读理解差"

这是一句正确的废话

三重表征指导后

同一个 L4，分别推 I0(符号) / I1(图示) / I1(文字)

精准定位：符号→数量通道OK，语词→数量通道断裂

四、双系统理论 → 学生状态表新增"流利度"维度

Q3 升维二：卡尼曼的双系统理论（Dual Process, 2011诺奖）

落地困境：学生状态表用 mastery(0~1) 追踪掌握概率。但有一类"隐性未掌握"传统方式抓不到——答对了，但磨蹭了 30 秒。

双系统如何改变决策：

  • 系统1(自动化,极速) vs 系统2(逻辑分析,慢)。真正掌握 = 知识从系统2迁移到系统1
  • 直接改变了 学生状态表的 Schema：在 mastery 之外新增 fluency_ms（平均作答时长）和 system1_ratio（自动化程度）
  • AI 推题策略升级：即使 mastery=0.95 但 fluency>15s → 不跳过，推送抗遗忘流利度训练直到 3 秒内答出

落地影响：学生状态表 V2.0

{
  "student_id": "S_001",
  "node_states": {
    "两位数加两位数(进位)": {
      "mastery": 0.95,        // BKT 掌握概率 ← SOLO 理论
      "fluency_ms": 28000,    // 平均作答时长 ← 双系统理论(新增!)
      "system1": false,       // 尚未自动化 → 仍需训练
      "prior_bias": null      // 无错误先验 ← 预测编码理论(新增!)
    }
  }
}

五、预测编码理论 → 考法图谱新增"先验偏差"列

Q3 升维三：弗里斯顿的预测编码（Free Energy Principle, 2010s）

落地困境：降维诊断的假设是"错=不会，降维找断点"。但有一类错误不是不会，是脑子里装了错误的规律。比如学生坚信"乘法让数字变大"→ 做 0.5×0.8 时判断结果变大。

预测编码如何改变决策：

  • 错误不是无能，是"预测误差"。学习=更新大脑的生成模型以最小化惊奇感
  • 直接改变了 ❹ 考法图谱：在"核心挖坑维度"之外，新增 "先验偏差(Prior Bias)"列——标注学生可能携带的错误先验模型
  • 改变了 AI 的纠错策略：遇到先验偏差类错误，不降维（降维没用，低级题也会被同一个错误先验支配），而是推送"违背直觉的最小特例"来打破先验
学生做 0.5×0.8=0.4 但判断"变大了"
↓ 传统降维：推送 5×8=40 → 做对 → 误判为"小数乘法不会"
实际错因：先验偏差"乘法=变大"← 整数经验的错误泛化
↓ 预测编码指导的新策略
推送最小特例："0.8块蛋糕的一半，你变多了还是变少了？"
→ 强烈预测误差 → 迫使大脑重写底层模型

六、LLM 原生分类 → 双轨制的"AI 体系"侧

Q4-Q5 的理论：CoT 推理深度 + Attention 锚点 + 双轨制

落地困境：教研员用 SOLO/三重表征打的标签，大模型看不懂。大模型不理解"L3 关联结构"，它只理解"需要几步推理链"。

双轨制如何改变决策：

  • 教研体系（人看）和 AI 体系（机器看）彻底解耦，中间用映射引擎自动转译
  • 落地到 题目 Q-Matrix：每道题同时有两套标签——教研标签（SOLO层级/情境类型/错因描述）和 AI 标签（CoT_Depth/Attention_Anchor/Error_Code）
  • 落地到 大模型的 System Prompt：当 CoT_Depth=Hop-3 时，强制要求模型输出至少3步 thinking → 幻觉率骤降

映射引擎：教研标签 → AI 标签的自动转译

教研体系（人看）映射规则AI 体系（机器看）
SOLO L1 单点→CoT_Depth: Hop-0 直接检索
SOLO L2 多点→CoT_Depth: Hop-1 单步映射
SOLO L3 关联→CoT_Depth: Hop-2 显式多步推理
SOLO L4 强关联→CoT_Depth: Hop-3 + Attention: A3_降噪

教研错因描述→RLHF Error Code（Err-E/Err-S/Err-B）

七、最近发展区 → 推题策略的"只推 L+1"铁律

Q2 的理论：维果茨基最近发展区（ZPD, 1930s）

落地困境：AI 该推什么难度的题？传统做法是"做对了就加难度"，线性且粗暴。

ZPD 如何改变决策：

  • ZPD 定义了"舒适区-发展区-恐慌区"三层结构
  • 直接催生了三大自适应场景的核心铁律：永远只推"当前稳定层级 + 1"
  • 落地到考法字典的四个维度——每次只升一个维度的一级：I0→I1（不跳I2），D1→D2（不跳D3）

溯源降级 ← ZPD + SOLO

做错 → SOLO 降维找断点 → 找到能做对的层级 → 从那里开始 ZPD L+1 重建

思想字典支持跨年级降维：六年级 ID-102 报错 → 直接推三年级同ID题

进阶跃迁 ← ZPD + 认知负荷

秒答全对 → 升一个I级(增加外在负荷) 或升一个D级(增加推理深度)

切换情境库引擎：标准→创新，保持知识不变但提升新鲜感
