## 一、项目背景

海豚 AI 学的教研团队需要将大量数学试卷录入到 Armory 后台系统，传统人工逐题录入效率极低（一套 30 题的试卷需要 2-3 小时）。丁晓冬发起了「后台自动录题工具人」项目，目标是用 AI 构建一套从**试卷解析 → 质量审核 → 预览确认 → 自动录入**的全链路自动化流水线，大幅提升录题效率。

## 三、四步流水线详解

### 步骤一：试卷转 JSON（解析层）

**成果**：开发了 `pdf-to-json` 解析技能，支持三种输入路径：

| 路径 | 输入格式 | 处理方式 |
|------|---------|---------|
| 路径 A | docx（公式已是 LaTeX） | python-docx 直接文本提取 |
| 路径 B | docx（含 MathType 二进制公式） | PDF 辅助 + AI 视觉识别 |
| 路径 C | 纯扫描 PDF / 图片 | 整页转图 + AI 视觉识别 |

**已完成的 7 套试卷**：

| 序号 | 试卷 | 来源格式 | 题目数 | 模块数 | 覆盖题型 |
|------|------|---------|-------|-------|---------|
| A | 山西运城芮城小升初数学 | PDF | 33 题 | 6 模块 | 选择/填空/解答/判断 |
| B | 河南洛阳伊川县小升初数学 | PDF | 24 题 | 5 模块 | 选择/填空/解答/计算 |
| C | 广东佛山八年级月考数学 | docx | 23 题 | 3 模块 | 选择/填空/解答（含图片） |
| D | 云南昆明西山区四年级期末数学 | Markdown | 28 题 | 5 模块 | 选择/填空/解答/判断 |
| E | 广西南宁江南区五年级期末数学 | Markdown | 30 题 | 6 模块 | 选择/填空/解答/计算 |
| F | 北京海淀小升初数学 | PDF | 24 题 | 4 模块 | 选择/填空/解答 |
| G | 人教版二年级数学上册期中测试卷 | — | 16 题 | 6 模块 | 选择/填空/判断/计算 |
| | **合计** | | **178 题** | | |

**关键技术突破**：
- 图片提取采用 PyMuPDF 嵌入式原图提取（无损），严禁截图裁剪
- 建立了统一的 Standard Question JSON Schema 作为全链路数据契约
- 支持内联图片 `{{IMG:url}}` 和精确尺寸控制 `{{IMG:url|width=W|height=H}}`

### 步骤二：试卷预览（校验层）

**成果**：开发了「录题预览」MiniApp，提供可视化校验能力。

**核心功能**：
- LaTeX 公式实时渲染预览，所见即所得
- 自动关联工作区内的 JSON 文件
- 三级校验分级：ERROR（必须修复）→ WARNING（建议确认）→ INFO（纯信息）
- 支持逐题浏览和快速定位问题题目

### 步骤三：自动审核（质量闸门）

**成果**：开发了 `question-review` 审核技能（当前 v1.4），包含 **100+ 条自动审核规则**，覆盖 6 大类检查：

| 审核类别 | 规则数 | 示例 |
|---------|-------|------|
| LaTeX 格式校验 | 10 条 | 分数必须 `\dfrac`、禁止裸 `%`、中文不入公式 |
| 结构完整性 | 7 条 | 题型映射合法、填空必须是数组、自动批改必须有展示答案 |
| 内容合理性 | 5 条 | 选择题答案在范围内、填空数量一致、多答案格式正确 |
| 公式丢失检测 | 3 条 | MathType 提取后空白检测、嵌入图公式识别 |
| 图片规范 | 4 条 | URL 格式、image_urls 为对象非数组、尺寸合理 |
| 数学计算验证 | 可扩展 | 答案与题干的数学一致性 |

**关键设计**：ERROR 级别问题自动修复后输出审核报告，用户确认后才能进入录入环节——**禁止跳过审核**。

### 步骤四：Armory 自动录入（录入层）

**成果**：开发了 `armory-auto-entry` Chrome MV3 扩展，实现 Armory 后台的批量自动录入。

**支持的录入能力**：

| 能力 | 说明 |
|------|------|
| **7 种题型** | 单选、多选、填空(自动批改)、填空(人工批改)、判断、计算、解答 |
| **ProseMirror 写入** | 通过粘贴事件模拟写入富文本编辑器，公式 `$...$` 自动渲染 |
| **选项自动补齐** | Armory 默认只有 A/B 两个选项，脚本自动点击「增加选项」补齐 C/D |
| **图片自动插入** | Canvas 预缩放（40%，最大 350px）→ paste File 事件插入 |
| **填空按钮插入** | 自动在 `{{BLANK:n}}` 位置插入填空组件 |
| **展示答案填写** | 填空(自动批改)每个空的展示答案编辑器自动填写 |
| **多答案录入** | 支持 `##` 分隔的多答案、匹配模式选择（任意/完全/保证顺序） |
| **系统查重** | 题干写入后自动检测重题，命中则一键复用 |
| **写入验证** | 每次写入后回读编辑器内容对比验证，失败自动重试 |
| **批量模式** | 支持 `run()` 连续录入多题，含进度统计 |
| **模块化录入** | v1.5.0+ 支持按模块分组录入 |

**架构亮点**：
- 双世界注入（Isolated World + MAIN World），兼容 Angular 变更检测
- 轮询/观察者模式等待 DOM 就绪，杜绝 `setTimeout` 盲等
- 每步异常自动暂停并输出状态，不静默跳过

## 四、规范体系沉淀

### 题型映射规范（与教研老师共同确认）

2026-04-10 由余老师和马老师最终确认的题型→录入形式映射表：

| 学科题型 | 录入形式 | 关键规则 |
|---------|---------|---------|
| 单选题 | 单选 | 选项独立存 options 数组 |
| 多选题 | 填空 | 选项放入题干 |
| 填空题 | 填空(自动/人工) / 解答 | 按答案内容判自动/人工；无法插填空按钮时用解答 |
| 判断题 | 判断 | 答案 true/false |
| 计算题（口算） | 填空 | 有分数→人工批改 |
| 计算题（非口算） | 解答 | 脱式计算、方程等 |
| 应用题/证明题等 | 解答 | 统一解答形式 |

**核心原则**：海豚不使用「多问大题」录入形式。含多小题的大题——全是填空则合并为一个填空，否则整题录为解答。

### LaTeX 格式规范（10 条铁律）

1. 分数**必须**用 `$\dfrac{a}{b}$`，严禁 `1/2`、`\frac`、`\over`
2. 括号用 `\left(\right)`
3. 汉字不入公式
4. □ 直接输入，不用 `\square`
5. 省略号用 `\cdots\cdots`
6. 循环小数用 `\overset{\bullet}{3}`
7. 百分号在 LaTeX 中用 `\%`
8. 平方/立方用 `^{2}` `^{3}`
9. 方程求解用 `aligned` 环境等号对齐
10. 每道题的 stem/answer/analysis 全部过自动校验

### 填空答案处理规范

- **批改答案 vs 展示答案**：明确区分系统判分用的答案和展示给学生的答案
- **多答案匹配**：任意匹配（默认）、完全匹配、保证顺序三种模式
- **答案符号转移**：百分号、单位等转移到题干，答案只填核心内容
- **跨空耦合检测**：多空多答案组合问题必须用人工批改

## 五、通用架构方案（可复用）

从项目实践中提炼出**平台无关的三层流水线架构**，设计了标准化的 `PlatformAdapter` 接口，可复用到任何题库系统：

```
Layer 1: 解析层（PDF/docx/图片 → Standard JSON）
    ↓
Layer 2: 校验层（Standard JSON → Validated JSON + 审计报告）
    ↓
Layer 3: 录入层（Validated JSON → 目标平台，通过 Adapter 适配）
```

**已验证的平台适配器**：

| 平台 | 适配方式 | 状态 |
|------|---------|------|
| Armory | UI 自动化（Chrome 扩展） | 生产使用中 |
| Cyber | API 直连 | 已验证 |
| Aries | MCP API | 已验证 |

**10 个经验教训**（踩坑清单）也已总结归档，涵盖 ProseMirror 写入、图片处理、LaTeX 渲染、登录态管理等核心问题。

## 九、技术创新点

1. **ProseMirror 粘贴写入方案**：解决了富文本编辑器无法用 innerHTML 赋值的行业通病，通过 ClipboardEvent 模拟实现可靠写入
2. **Canvas 预缩放图片方案**：解决了 ProseMirror 不保留 width 属性的问题，上传前在客户端完成尺寸调整
3. **双世界注入架构**：Chrome MV3 的 Isolated World + MAIN World 注入，兼容 Angular 变更检测
4. **Standard JSON Schema**：设计了平台无关的题目数据格式，使解析结果可以对接任何题库系统
5. **强制审核闸门**：在流水线中内嵌质量关卡，任何 JSON 必须过审才能录入，从机制上杜绝低质量数据入库
